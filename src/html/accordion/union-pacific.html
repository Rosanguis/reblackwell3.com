• What I Do

    Simply put, I write software that helps the railroad function.

    To better understand my role as a Java developer within NetControl's Equipment division, a short history lesson is in order. In the 1960's, programmers built Transportation Control System in COBOL (a programming language that is no longer prevalent) and ran it on a mainframe. Nearly 40 years later, the decision was made to replace this mainframe program written in an outdated language with a modern program written in Java that can run on a distributed computer system instead of an expensive mainframe. NetControl, as the project is called, is a $200 million project that has been active since 2006. It was originally intended to be a 10 year project.

    NetControl Equipment has the responsibility of recreating the software pertaining to equipment (i.e. railcars and their contents) that exists in the 1960's COBOL code. The software modules created by the equipment group are integral to the larger NetControl project.

    Specifically addressing my role within Union Pacific Railroad, I use Java to build micro-services that connect to other micro-services in NetControl Equipment to accomplish complex tasks. NetControl is designed as an intricate web of services that each do one specific task and all combine to perform larger tasks.

  • How I Do It

    I use Java and a suite of tools surrounding it including Spring, Maven, Subversion, and JUnit.

    NetControl makes an effort to provide a relatively standardized development environment to reduce friction within the project.

    First, we employ the Agile model to plan ahead and manage our expected workload. On a more technical note, we use Maven to standardize our build process and manage dependencies and Spring for dependency injection. Our project uses Subversion for source control. We are also sure to thoroughly unit test our programs, and we use JUnit for the testing and JMock for mocking classes.

    I am very pleased by the fact that Union Pacific aims to improve the quality of its code and developers by frequently having code reviews. As a new developer, code reviews have been an excellent resource to quickly learn industry standards and finding ways to write more readable code.
